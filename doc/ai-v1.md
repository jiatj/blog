请实现一个个人网站 V1，定位是“个人入口 + 工具发布站 + 博客发布层”。

【项目目标】
这个网站服务于个人 IP（AI Builder Lab），当前主要用途：
1. 作为正式网站入口
2. 展示和发布工具
3. 发布博客内容
4. 支持未来升级，但当前不做平台化

【核心原则】
- 极简、克制、精致
- 不炫技，不堆功能，不做复杂营销页
- 重视秩序感、留白、排版和整体一致性
- 优先可用、可维护、可继续被 AI 修改
- 先完成 V1，不做过度预埋

【技术要求】
- 使用 Next.js
- 使用 TypeScript
- 使用 Tailwind CSS
- 可以使用 shadcn/ui
- 支持 Markdown / MDX 内容渲染
- 支持 i18n
- 支持 Light / Dark theme
- 支持 SEO
- 不使用数据库
- 不引入重型 CMS
- 不做后台系统
- 不做用户系统
- 不做支付、评论、推荐等复杂功能

【内容来源】
- 内容来自本地 Markdown / MDX 文件
- 上游写作工具默认是 Obsidian
- 网站只负责消费内容并发布
- 代码结构上要让未来替换内容源成为可能，但当前只实现本地文件驱动

【V1 必做页面】
1. 首页
2. 工具列表页
3. 工具详情页
4. 博客列表页
5. 博客详情页
6. About / AI Builder Lab 页面
7. 404 页面

【首页要求】
首页不是传统博客首页，也不是 SaaS 营销页。
它应该像：
- 个人品牌入口
- 工具导航入口
- 内容入口

首页内容保持克制，建议只包含：
- 简短介绍
- 工具入口
- 最新博客入口
- AI Builder Lab 简介
- 清晰导航

不要做成长篇叙事，不要堆很多区块。

【工具页要求】
工具列表页展示工具卡片。
每个工具至少包含：
- 名称
- 一句话简介
- 状态（Live / Beta / Draft）
- 跳转入口

工具详情页至少包含：
- 名称
- 价值说明
- 功能介绍
- 使用入口
- 可选 GitHub / 更新记录 / 截图

【博客要求】
博客列表页展示：
- 标题
- 摘要
- 日期
- 标签

博客详情页重点是阅读体验：
- 字号、行高、间距舒适
- 标题层级清楚
- 代码块样式简洁
- 移动端也要好读

【内容模型】
至少支持两类内容：

Post:
- title
- slug
- date
- summary
- tags
- locale
- draft
- cover
- seoTitle
- seoDescription

Tool:
- title
- slug
- summary
- locale
- draft
- toolStatus
- toolUrl
- repoUrl
- cover
- seoTitle
- seoDescription

请基于 frontmatter 读取和组织内容，不要把内容结构写死在页面里。

【SEO 要求】
V1 必须支持基础 SEO：
- 每个页面有独立 title 和 description
- 博客和工具详情页支持 frontmatter SEO 字段
- URL 清晰
- 支持 sitemap
- 支持 robots.txt
- 支持 Open Graph 基础信息
- 页面使用语义化 HTML

【i18n 要求】
- 支持中英文
- 支持 locale 路由
- UI 文案必须可国际化
- 内容允许分别维护中英文
- 不要求所有文章双语同步
- 国际化实现保持轻量，不炫技

【主题要求】
- 支持 Light / Dark
- 风格要安静、克制、精致
- 不要复杂主题系统
- 不要为了灵活性做大量主题抽象
- 如果你认为可行，可以预留少量主题变量，但不要做复杂皮肤系统

【视觉要求】
整体视觉必须：
- 极简
- 安静
- 留白充足
- 层级分明
- 精致但不浮夸

禁止：
- 赛博朋克式视觉噪音
- 过度动画
- 杂乱卡片堆叠
- 高饱和装饰色泛滥
- 看起来像模板站或营销站

页面要像一个长期会使用的个人网站，而不是短期宣传页。

【工程要求】
- 项目结构清晰
- 内容读取逻辑集中管理
- i18n 逻辑集中管理
- SEO 逻辑集中管理
- 样式变量尽量集中管理
- 组件边界清楚
- 避免过度抽象
- 避免复杂状态管理
- 优先静态生成思路
- 代码要方便后续继续被 AI 修改

【不要做的事】
- 不要为了“以后可能用到”而提前做复杂系统
- 不要引入数据库
- 不要做后台
- 不要做复杂插件机制
- 不要做复杂主题市场
- 不要过度封装
- 不要做平台化设计

【你的输出要求】
请直接完成一个可运行的 V1 项目实现方案，并优先保证：
1. 页面完整可用
2. 风格统一
3. 结构清晰
4. 内容链路稳定
5. 后续容易迭代

如果遇到实现选择，优先选择：
- 更简单
- 更稳定
- 更清晰
- 更容易被 AI 后续修改

不要追求炫技和大而全。
