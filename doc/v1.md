# AI Builder Lab 网站 V1 PRD

## 1. 项目定位

这是一个 **个人网站入口 + 工具发布站 + 博客发布层**。  
V1 的目标不是做完整内容平台，也不是做 CMS，而是先形成一个**可正式上线、可持续迭代、可承载个人 IP 与工具入口**的网站。

内容生产方式以 **本地 Obsidian 写作** 为主，网站只负责消费内容并发布。

---

## 2. V1 目标

V1 只解决四件事：

1. 形成一个正式可用的网站入口
    
2. 支持发布工具介绍页
    
3. 支持发布博客文章
    
4. 为未来升级保留清晰结构，但**不提前引入数据库和复杂后台**
    

---

## 3. 非目标

V1 明确 **不做**：

- 不做数据库
    
- 不做用户系统
    
- 不做评论
    
- 不做支付/会员
    
- 不做后台在线编辑
    
- 不做复杂主题系统
    
- 不做内容推荐系统
    
- 不做搜索增强以外的复杂信息架构
    
- 不做“平台化”能力
    

---

## 4. 产品原则

### 4.1 总原则

**极简、克制、精致。**

页面必须有高级感，但不能靠复杂装饰堆砌。  
重点不是“设计感很强”，而是“秩序感很强”。

### 4.2 信息原则

- 首页信息密度适中
    
- 导航层级尽量少
    
- 每个页面只突出一个核心目标
    
- 内容优先，装饰退后
    
- 文案尽量短，避免喧闹
    

### 4.3 工程原则

- 无数据库，文件驱动
    
- 结构清晰，便于 AI 修改
    
- 组件可复用
    
- 内容源与展示层解耦
    
- V1 优先静态生成与 SEO
    

---

## 5. 目标用户

当前主要是三类访问者：

1. 通过内容认识你的读者
    
2. 想了解你发布的工具的人
    
3. 想理解 AI Builder Lab 方向的人
    

V1 不考虑复杂用户分层，不做个性化体验。

---

## 6. 内容来源与发布方式

### 6.1 内容源

- 博客、工具说明、部分固定页面内容来自本地 Markdown / MDX 文件
    
- 上游写作工具默认是 **Obsidian**
    
- 未来可以兼容其他本地写作工具，只要能输出相同格式
    

### 6.2 发布方式

- 本地内容进入项目内容目录后构建发布
    
- 网站读取 frontmatter 与正文内容生成页面
    
- V1 以静态站方式输出优先
    

### 6.3 内容规范

必须通过统一 frontmatter 组织内容。  
禁止把页面逻辑硬编码在文章内容里。

---

## 7. 功能范围

## 7.1 页面结构

V1 至少包含以下页面：

- 首页
    
- 工具列表页
    
- 工具详情页
    
- 博客列表页
    
- 博客详情页
    
- About / AI Builder Lab 页面
    
- 404 页面
    

### 7.2 首页

首页是网站入口，不做复杂叙事。  
建议只承担以下功能：

- 简短介绍你是谁 / 在做什么
    
- 工具入口
    
- 最新内容入口
    
- AI Builder Lab 简介
    
- 清晰导航
    

首页不能做成资讯门户，也不能做成长篇品牌宣言。

### 7.3 工具列表页

用于展示你发布的工具。  
每个工具卡片至少包含：

- 名称
    
- 一句话简介
    
- 状态（如 Live / Beta / Draft）
    
- 链接入口
    

### 7.4 工具详情页

用于承载单个工具的独立说明。  
至少包含：

- 工具名称
    
- 一句话价值说明
    
- 功能简介
    
- 使用入口
    
- 可选：GitHub / 更新记录 / 截图
    

### 7.5 博客列表页

展示文章列表，支持基本分类信息：

- 标题
    
- 摘要
    
- 日期
    
- 标签
    

V1 不要求复杂筛选系统。

### 7.6 博客详情页

重点是阅读体验。  
要求：

- 字号、行高、留白舒适
    
- 标题层级清楚
    
- 代码块和引用块样式简洁
    
- 移动端阅读也自然
    

### 7.7 About 页面

用于说明：

- 你是谁
    
- AI Builder Lab 是什么
    
- 网站的主要方向
    
- 联系方式 / 外链
    

---

## 8. SEO 要求

V1 必须支持基础 SEO。  
要求：

- 所有页面有独立 title 和 description
    
- 博客与工具详情页支持基于 frontmatter 的 SEO 字段
    
- 规范化 URL
    
- 生成 sitemap
    
- 生成 robots.txt
    
- 支持 Open Graph 基础信息
    
- 博客文章页具备良好的语义化结构
    

SEO 是 V1 必做项，不允许后补。

---

## 9. i18n 要求

V1 支持国际化，但采取**轻量策略**。

### 9.1 范围

- UI 文案必须支持中英文
    
- 页面路由支持 locale
    
- 内容层允许中英文分别维护
    
- 不要求所有文章全部双语同步
    

### 9.2 要求

- 路由结构清晰
    
- 默认语言明确
    
- 切换逻辑简单，不炫技
    
- 不做复杂自动翻译链路
    

---

## 10. 主题与视觉要求

## 10.1 视觉原则

整体风格必须：

- 克制
    
- 安静
    
- 精致
    
- 有秩序
    
- 不浮夸
    

### 10.2 禁止项

禁止出现以下倾向：

- 大面积炫技动画
    
- 过重拟物
    
- 赛博朋克式噪音视觉
    
- 过多高饱和装饰色
    
- 杂乱卡片堆叠
    
- “产品很多所以首页很满”的布局
    

### 10.3 主题要求

V1 支持：

- Light
    
- Dark
    

可选预留一个品牌倾向主题，但不是必须。

### 10.4 第三方主题

可以参考第三方主题，但**不以接入第三方主题为目标**。  
主题系统不能复杂化。  
如果第三方主题会破坏整体一致性、增加维护负担或降低 AI 可修改性，则放弃。

---

## 11. 设计与布局约束

这是给 AI 编码特别重要的一段。

### 11.1 布局要求

- 页面宽度控制克制
    
- 留白充足
    
- 模块数少而明确
    
- 卡片数量适度
    
- 层级清楚，避免信息噪音
    

### 11.2 视觉气质

不是“炫”，而是“稳”。  
不是“功能很多”，而是“少而准”。  
不是“设计师作品集感”，而是“长期可使用的个人网站感”。

### 11.3 首页要求

首页不能像 SaaS 营销页。  
也不能像传统博客模板。  
更接近：**个人品牌入口 + 工具导航 + 内容入口。**

### 11.4 字体与排版

- 以可读性优先
    
- 标题层级要明显
    
- 正文排版要舒适
    
- 禁止密集、压迫、炫耀式设计

## 13. 技术方向

V1 技术方向固定如下：

- Next.js
    
- TypeScript
    
- Tailwind CSS
    
- shadcn/ui
    
- i18n 方案
    
- 主题切换方案
    
- Markdown / MDX 内容消费
    

### 约束

- 不引入数据库
    
- 不引入重型 CMS
    
- 不引入不必要的服务端复杂逻辑
    
- 不引入复杂状态管理
    

---

## 14. AI 编码约束

这一段是给 Codex / AI 非常重要的。

### 14.1 优先级

AI 编码时优先保证：

1. 结构清晰
    
2. 页面可用
    
3. 样式克制统一
    
4. 内容链路稳定
    
5. 后续易改
    

### 14.2 不要做的事

- 不要为了“完整”提前造后台
    
- 不要提前抽象过度
    
- 不要做复杂插件系统
    
- 不要过度封装主题
    
- 不要为了灵活而牺牲可读性
    
- 不要做大量未来也许会用到的预埋功能
    

### 14.3 代码要求

- 组件边界清晰
    
- 内容读取逻辑集中管理
    
- SEO 逻辑统一组织
    
- i18n 逻辑统一组织
    
- 样式变量尽量集中
    
- 可静态生成优先
    

### 14.4 UI 要求

- 简洁
    
- 高质量留白
    
- 层级清晰
    
- 少即是多
    
- 精致但不装饰化
    

---

## 15. V1 验收标准

V1 完成时，至少满足：

1. 网站可正式访问
    
2. 首页可作为统一入口
    
3. 工具列表与详情页可用
    
4. 博客列表与详情页可用
    
5. Obsidian 内容可发布
    
6. 支持中英文路由与基本文案切换
    
7. 支持 Light / Dark 主题
    
8. SEO 基础能力完整
    
9. 页面整体视觉克制、统一、精致
    
10. 项目结构清晰，后续可升级
    

---

## 16. V1 成功标准

V1 的成功不看功能数量，主要看三点：

1. **你能稳定发布内容和工具**
    
2. **这个站已经足够作为正式入口使用**
    
3. **后续继续升级时，不需要推翻重来**
